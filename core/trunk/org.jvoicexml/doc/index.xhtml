The system was tested before the release, but clearly you can never test enough. If any problems should occur, please submit bug reports via the Sourceforge bugtracking system.

Mailing lists

The best way for communication is to subscribe to our mailing lists.

If you want to discuss the coding, make suggestions for improvement or if you have trouble building the binaries, use the developer list.

If you want to get help on our API for your current project, use the user list.

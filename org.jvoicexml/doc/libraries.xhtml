Library dependencies

JVoiceXML libraries

JVoiceXML comprises the following jar files:

  • org.jvoicexml.jar
    This is the main jar, containing the VoiceXML interpreter
  • org.jvoicexml.callmanager.mmi.jar
    CallManager implementation with support for the W3C MMI architectural pattern
  • org.jvoicexml.callmanager.mmi.http.jar
    HTTP ETL for the MMI Callmanager
  • org.jvoicexml.callmanager.mmi.socket.jar
    Socket ETL for the MMI Callmanager
  • org.jvoicexml.callmanager.mmi.umundo.jar
    umundo ETL for the MMI Callmanager
  • org.jvoicexml.callmanager.sip.jar
    CallManager implementation with support for SIP (unfinished)
  • org.jvoicexml.callmanager.text.jar
    CallManager implementation for text based access (unfinished)
  • org.jvoicexml.client.jar
    jar containing the interfaces to create client applications.
  • org.jvoicexml.config.jar
    spring based configuration of JVoiceXML
  • org.jvoicexml.interpreter.datamodel.ecmascript.jar
    This jar contains a datamodel based on ECMAScript
  • org.jvoicexml.implementation.jsapi10.jar
    This jar contains a basic implementation platform based on the JSAPI 1.0.
  • org.jvoicexml.implementation.jsapi20.jar
    This jar contains an unfinished implementation platform based on the JSAPI 2.0.
  • org.jvoicexml.implementation.kinect.jar
    This jar contains an unfinished implementation platform for the kinect.
  • org.jvoicexml.implementation.lightweightbml.jar
    This jar contains a basic implementation platform for an Avatar that supports BML.
  • org.jvoicexml.implementation.marc.jar
    This jar contains support for MARC
  • org.jvoicexml.implementation.mary.jar
    This jar contains support for the OpenMary TTS engine.
  • org.jvoicexml.implementation.text.jar
    This jar contains a basic implementation platform for text based interaction.
  • org.jvoicexml.jndi.jar
    This jar contains the JNDI support for JVoiceXML which is needed for remote access.
  • org.jvoicexml.mmi.events.jar
    Class library for authoring MMI events.
  • org.jvoicexml.processor.srgs.jar
    This jar contains an SRGS grammar processor.
  • org.jvoicexml.xml.jar
    This jar contains all the XML related stuff
  • org.jvoicexml.client.text.jar
    This jar containing the classes to access the text based implementation platform
  • org.jvoicexml.implementation.jtapi.jar
    This jar contains an unfinished implementation for jtapi 1.3.1 support
  • org.jvoicexml.implementation.mrcpv2.jar
    This jar contains a basic implementation platform for MRCPv2.
  • org.jvoicexml.talkinghead.jar
    This jar contains the talking head implementation
  • org.jvoicexml.voicexmlunit.jar
    Unit testing framework for VoiceXML.

Library dependencies

All libraries can be found in the $JVOICEXML_HOME/lib folder. You will need:

Create VoiceXML documents

  • org.jvoicexml.xml.jar

Run the interpreter

  • org.jvoicexml.jar
  • org.jvoicexml.xml.jar
  • org.jvoicexml.processor.srgs.jar
  • org.jvoicexml.client.jar
  • org.jvoicexml.jndi.jar (optional)
  • log4j 1.2.17 or above (http://logging.apache.org/log4j/)
  • commons libraries (http://jakarta.apache.org/commons/)
    • logging 1.1.1
    • lang 3.3.2
    • pool 1.5.5
    • http core 4.2.3
    • http client 4.2.3
    • codec 1.4
  • Jetty 9.2.2 http://download.eclipse.org/jetty/
  • spring framework 4.2.0 (http://www.springsource.org/)
  • jchardet 1.1 (http://jchardet.sourceforge.net/)

ECMAScript data model

  • rhino 1.7R4 (http://www.mozilla.org/rhino/)
  • json-simple 1.1 (http://code.google.com/p/json-simple/)

Interpreter with JSAPI 1.0 support

  • org.jvoicexml.implementation.jsapi10.jar

Interpreter with JSAPI 2.0 support

  • org.jvoicexml.implementation.jsapi20.jar
  • jsr113.jar
  • org.jvoicexml.jsapi2.jar

  • sphinx 4 (SVN snapshot from 12/12/2014, http://cmusphinx.sourceforge.net/)

Interpreter with Kinect support

  • org.jvoicexml.implementation.kinect.jar

Interpreter with Lightweight BML

  • org.jvoicexml.implementation.lightweightbml.jar
  • jdom 2.0.5

Interpreter with Mary support

  • marytts-client-5.2-SNAPSHOT-jar-with-dependencies.jar (modified according to pull request #227) http://mary.dfki.de)

Interpreter with MARC support

Interpreter with MRCPv2 support

  • org.jvoicexml.implementation.mrcpv2.jar
  • cairo-client.jar

  • MRCPv2 server

Interpreter with text based interface

  • org.jvoicexml.text.jar

MMI

  • org.jvoicexml.callmanager.mmi.jar
  • org.jvoicexml.mmi.events.jar
  • org.jvoicexml.callmanager.mmi.servlet.war

The war archive can be found in the lib folder of your installation and must be deployed to a servlet container if you want to use VoiceXML snippets in the content attributes.

HTTP ETL for the MMI callmanager

  • org.jvoicexml.callmanager.mmi.http.jar
  • Jetty 9.2.2 http://download.eclipse.org/jetty/

Socket ETL for the MMI callmanager

  • org.jvoicexml.callmanager.mmi.socket.jar

umundo ETL for the MMI callmanager

  • org.jvoicexml.callmanager.mmi.umundo.jar
  • umundo.jar (https://github.xom/tklab-tud/umundo)
  • protobuf-java-2.5.0.jar ( http://code.google.com/p/protobuf)

Run a client

  • org.jvoicexml.client.jar
  • org.jvoicexml.xml.jar (optional if you want to author VoiceXML documents
  • org.jvoicexml.jndi.jar (optional if you want to access JVoiceXML remotely
  • org.jvoicexml.client.text.jar (optional, to access the text based implementation platform)

You can also download the third party libraries from their web sites.

Create VoiceML unit tests

JVoiceXML must be configured to use the text implementation platform.

  • org.jvoicexml.voicexmlunit.jar
  • org.jvoicexml.client.jar
  • org.jvoicexml.client.text.jar
  • junit.jar (http://junit.sourceforge.org)

Talking Head

JVoiceXML must be configured to use the lightweight BML implementation platform

  • jdom 2.0.5
  • marytts-client-5.2-SNAPSHOT-jar-with-dependencies.jar (modified according to pull request #227) http://mary.dfki.de)

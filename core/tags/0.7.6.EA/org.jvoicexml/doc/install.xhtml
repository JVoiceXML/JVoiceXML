Installation of JVoiceXML

Requirements

JVoiceXML uses features of J2SE 1.6 and will not run with versions below. Optional: ANT 1.7 (http://ant.apache.org) to run the demos.

Talking Java implementation platform

Thanks to Jontahan Kinnersley, JVoiceXML ships with the Talking Java hook from cloudgarden. This copy is free for private use. You should buy a license if you use it in a commercial setting.

JSAPI 2 implementation plaform

This is an implementation of a draft specification developed under the Java Community Process (JCP) and is made available for testing and evaluation purposes only. The code is not compatible with any specification of the JCP.

Marc implementation platform

This implementation platform requires an installation of MARC and OpenMary. MARC is a talking head that can be be downloaded from http://marmarc.limsi.fr. In addition you need to have an installation of OpenMary which can be downloaded from http://mary.dfki.de. You will also need to configure MARC with the location of your OpenMary installation. This implementation platform can be used as a replacement for any TTS engine. Note, that the use of this platform requires the use of an ASR of another implementation platform.

Mary implementation platform

This implementation platform requires an installation of OpenMay which can be downloaded from http://mary.dfki.de. This implementation platform can be used as a replacement for any TTS engine. Note, that the use of this platform requires the use of an ASR of another implementation platform.

JVoiceXML must be called with a SIP phone. We used the xlite phone which can be downloaded for free from http://www.counterpath.com

MRCPv2 implementation platform

This implementation platform requires the presence of an MRCPv2 server. An open source solution based on FreeTTS and sphinx 4 can be downloaded from http://cairo.sourceforge.net.

JVoiceXML must be called with a SIP phone. We used the xlite phone which can be downloaded for free from http://www.counterpath.com

JVoiceXML Installation

Using the installer (Windows and Linux)

Simply unzip jvxml-<version>.zip to extract the installer jar archive. Become root on Linux machines.
Call
java -jar jvxml-<version>-install.jar
to install the JVoiceXML voice browser.

Alternatively, a double-click on the jar should perform the magic on Windows.

Using the rpm (Linux)

Become root and call
rpm -ivh jvoicexml-<version>.rpm

The installation folder is referred as JVOICEXML_HOME

Start the VoiceXML browser

On Linux

sh $JVOICEXML_HOME/bin/startup.sh

On Windows

Doubleclick: 
%JVOICEXML_HOME%/bin/JVoiceXML.exe
or 
%JVOICEXML_HOME%/bin/startup.bat

Test the VoiceXML browser

JVoiceXML comes with a couple of demos, which can be used to test the browser.

To run the demos, copy the file ant.properties from the folder $JVOICEXML_HOME/demos/config-props to the folder $JVOICEXML_HOME/demos/personal-props and adapt the settings in the copied file

In most cases it should be sufficient to change to each demo directory and call

ant run
